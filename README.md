## 🏗️ 三层架构设计

OctoWork Chat 采用创新的三层架构，确保系统稳定、安全且易于扩展：

```
用户电脑上的三层结构：

  ┌─────────────────────────────────────────────┐
  │  octowork-chat  （嘴）                        │
  │  用户对话 → AI 理解意图 → 调用工厂工作流       │
  └─────────────────────────────────────────────┘
                        ↓
  ┌─────────────────────────────────────────────┐
  │  .openclaw / OpenClaw  （手脚）               │
  │  AI 执行层，负责调用工具、跑脚本、处理文件     │
  └─────────────────────────────────────────────┘
                        ↓
  ┌─────────────────────────────────────────────┐
  │  octowork/  数字公寓  （家）                  │
  │  ├── company/  公司层                        │
  │  ├── departments/  部门层                    │
  │  ├── personal/  个人层                       │
  │  └── projects/OctoWork_VisualFactory         │
  │      内容能力中台（本文档描述的系统）           │
  └─────────────────────────────────────────────┘
```

**每一层的功能：**
1. **octowork-chat（嘴）**：用户界面层，你在这里与AI员工对话，下达指令
2. **OpenClaw（手脚）**：AI执行层，负责实际执行任务，调用工具和脚本
3. **octowork数字公寓（家）**：AI员工的生活和工作空间，包含完整的AI团队架构

## 🚀 快速开始

### 环境要求
- **操作系统**：macOS / Windows / Linux
- **Node.js**：预装 Node.js v18.0 或以上版本
- **网络**：稳定的互联网连接（用于AI模型调用）

### 📦 安装步骤

#### 1. 下载安装包
从官网下载最新版的 `octowork-chat-v1.1.5.zip` 安装包。

#### 2. 解压安装
将安装包解压到你的用户目录：
```bash
# macOS/Linux
unzip octowork-chat-v1.1.5.zip -d ~/octowork-chat

# Windows
# 右键解压到 C:\Users\你的用户名\octowork-chat
```

#### 3. 一键安装
运行安装脚本，自动完成所有配置：
```bash
# macOS/Linux
cd ~/octowork-chat
./install.sh

# Windows
# 双击运行 install.bat
```

**安装脚本会自动：**
- 检查并安装必要的依赖
- 配置运行环境
- 设置开机自启动（可选）

#### 4. 启动应用
```bash
# 方法一：手动启动
cd ~/octowork-chat
./start.sh

# 方法二：使用启动器（推荐）
双击 "Octowork启动器.command"（macOS/Linux）
或 "Octowork启动器.exe"（Windows）
```

#### 5. 访问应用
打开浏览器，访问：**http://localhost:1314**

### 🔑 首次激活

首次启动时，系统会显示你的**机器指纹**。将指纹发送给管理员，获取对应的 `license.key` 授权文件。

将 `license.key` 文件放在 `~/octowork-chat/` 目录下，重启应用即可完成激活。

## ⚙️ 系统配置

### 环境要求
确保以下目录结构存在：
```
~/
├── .openclaw/          # AI 执行引擎（安装脚本会自动创建）
├── octowork/           # 数字公寓（安装脚本会自动创建）
└── octowork-chat/      # 本程序（你安装的位置）
```

### 端口配置
- **服务端口**：1314（可在启动器菜单中修改）
- **如果端口被占用**：启动器会自动检测并提供解决方案

### 自动启动
安装时选择“配置自动启动”，系统会：
- 创建开机自启动项
- 设置每天自动重启服务（保持系统稳定）
- 自动监控服务健康状态

## 📞 支持与服务

### 遇到问题？
1. **查看日志**：`~/octowork-chat/backend/logs/`
2. **重启服务**：使用启动器的"重启服务"功能
3. **重新安装**：运行 `./install.sh --reinstall`

### 获取帮助
- **官方文档**：https://docs.octowork.ai
- **技术支持**：support@octowork.ai
- **社区讨论**：加入 OctoWork 用户社区

### 故障排除

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 无法启动 | 端口被占用 | 修改端口号或关闭占用程序 |
| 无法连接AI | 网络问题 | 检查网络连接，确认API密钥有效 |
| 授权失败 | license.key 无效 | 重新获取正确的授权文件 |
| 内存不足 | 同时运行太多AI任务 | 关闭不必要的AI员工 |
